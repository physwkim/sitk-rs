# Upstream findings ledger — ITK / SimpleITK defects and divergences

Central index of every upstream (ITK C++ / SimpleITK wrapping) defect,
inconsistency, or quirk discovered while porting, plus every place this port
deliberately diverges from upstream. **The authoritative text for each entry
is the module doc named in its "doc" field** — this file is the index, not
the source of truth. Keep both in sync when adding entries.

Porting policy (see per-module docs for the citations):

- **reproduced** — upstream behavior is reproduced bit-for-bit, quirk
  included, and pinned by a test. Parity wins over correctness.
- **diverged** — this port intentionally differs; the module doc states the
  divergence and the reason (memory safety, defined behavior for C++ UB, or
  f64 precision).
- **open** — a decision is pending on whether to keep parity or fix.
